## cap

The `cap` unit measures the distance from the baseline to the top of capital or uppercase letters. It is typically used in conjunction with other font-relative units.

This unit is useful for creating layouts that are scale-independent and responsive to different fonts and font sizes. For instance, you can use the `cap` unit to set the height of headings and other elements, ensuring that they maintain their visual proportions regardless of the font choice.

## ch

The `ch` unit represents the advance measure of the character `0` (zero) in the font used to render it. It is an absolute length unit, though it scales with the font-size of the current element.

The `ch` unit was mostly used for sizing elements in monospace fonts, where all characters have the same width. This is because the advance measure of the character `0` is consistent for all monospace fonts. But even without actually using it with a monospace font, we can create some lovely responsive behavior with it. A fun thing to note though is that this works in multiple writing modes: when the inline axis is vertical on a webpage, the calculation is based on the height of the zero glyph.

How about a one-liner responsive grid? [This little article says that the ideal reading length is between 50-60 characters](https://baymard.com/blog/line-length-readability) and I've read some other studies before in the past which gave it an average of about 45-75 characters. How about we play with that idea for a grid?

Letâ€™s say we only want a grid where items are only next to each other if they can at least fit 45 characters in one line. We could do this with CSS Grid and the `ch` unit

```css
.grid {
  --grid-min: 45ch;
  --grid-gap: 2vmax;
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(var(--grid-min), 100%), 1fr));
  gap: var(--grid-gap);
}
```

## em

The `em` unit - known by many - represents the current font size of the element it is applied to. It is a very versatile unit that can be used to define margins, paddings, font sizes, and other properties of an element relative to its parent element's `font-size`. Itâ€™s a very versatile unit that is easy to understand but can be hard to master.

I know many people who are strong in using the `em` unit (and I applaud them!), but I've always found the relative size based on the parent a hard thing to master. Unless you are designing straight inside the browser and you donâ€™t have a pixel-perfect designer looking over your shoulderâ€¦ ;) \
Joking aside, I do believe it is an important unit to get the hang of even though I might need some practice myself.

To show a special use case of this example, I thought the usage of media queries could be interesting.

Imagine the following scenario: We have 3 `<div>` elements inside of the HTML, each containing three children, we give those wrappers a class based on the media query unit weâ€™re going to trigger:

```html
<div class="items px">
  <div class="item"></div>
  <div class="item"></div>
  <div class="item"></div>
</div>
<div class="items em">
  <!-- three item divs -->
</div>
<div class="items rem">
  <!-- three item divs -->
</div>
```

Some basic styling aside, The items should switch to a three-column layout at a certain point. Important to note is that we havenâ€™t changed the default `font-size` of `1rem` (`16px` by default).

These are our media queries:

```css
@media (min-width: 640px) {
  .px {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 2vmax;
    background-color: hotpink;
  }
}

@media (min-width: 40em) {
  .em {
    /* Same as we did for px */
  }
}

@media (min-width: 40rem) {
  &.rem {
    /* Same as we did for px */
  }
}
```

Now we both have an equivalent of `640px` in the (r)em values (16x40 = 640) and by no surprise, with the basic font-size in your browser, all of these will trigger at the same time:

## rem and rlh

Most of us have used the `rem` unit before, it is short for "root em" and measures the font size of an element relative to the font size of the root element. The root element, typically represented by the `<html>` element, has a default font size of `16px` in most browsers. Therefore, `1rem` is equivalent to `16px` if not changed by a reset.

There are some front-end developers that swear by using rems as a value for media queries.

```css
@media (width <= 32rem) {
  h1 {
    font-size: 2rem;
  }
}
```

This takes some getting used to and actually still is the better metric for users that change their default font sizes. There are a few articles to be found on this subject â€œpixels vs (r)em for media queriesâ€, but most of them are a bit too opinionated for my taste to add in this article.

Speaking of root based units, we also have rlh, which is then based on the root `line-height`.
